⏱ Refresh Strategy — Every 2 Days
Impact API + Amazon CSV + URLGenius CSV all pull on the same 48-hour scheduler. One cron job, one processing script, one new knowledge block injected into the AI. Meta + SYL refresh weekly since they inform patterns not current deals. Walmart Direct API + Crawlfeeder are on-demand only — triggered when a specific product lookup is needed.
Layer 2 — Processing Pipeline

What Happens to the Data Before the AI Sees It

1

Ingest + Normalize

Python script reads all CSVs and API responses. Strips duplicates, normalizes product names, maps ASINs to product details via Crawlfeeder. Assigns a source tag to every record (urlgenius / amazon / walmart / syl).

4

Generate AI Knowledge Block

The script writes a structured text file (~3,000–4,000 tokens) containing: Top 20 scored products with name + affiliate link + price + source stats. Retailer routing table. Content hook patterns from Meta. This file is injected as the AI's knowledge base on each refresh.

Top 20 Products Routing Table Hook Patterns Category Leaders
5

Affiliate Link Resolution

Every product in the knowledge block needs a working affiliate link. Resolution order: (1) Use URLGenius short link if it exists (already tracked, has image). (2) Generate via Walmart Direct API for Walmart products. (3) Build Amazon link using ASIN + mommymedeals-20 tag. (4) Fall back to Impact-generated link for Target. (5) Use SYL URL for other retailers.

Layer 3 — AI Knowledge Base

What the AI Actually Knows

The knowledge base is a structured text block injected into the Claude system prompt every 2 days. It's organized into 6 sections so the AI can find the right signal for any question type.

🔥
Hot Products Right Now
Top 20 products by composite Hot Score. Includes name, price, affiliate link, click count, units sold, and a 1-line "why it's hot" note.
~800 tokens · refreshed every 2 days
🏆
Category Champions
Top 3 products per major category (Toys, Baby, Beauty, Home, Grocery, Clothing). Used when someone asks "what's the best [X]?"
~600 tokens · refreshed every 2 days
🏪
Retailer Routing Rules
For each category: which store to recommend first, why, and what the CVR/earnings data says. AI uses this to give smart retailer advice.
~300 tokens · refreshed weekly
📝
Content Hook Patterns
What works on Meta: price-shock formula ($1 clearance = 158K views), brand partner format, urgency triggers. AI uses these when drafting captions.
~400 tokens · refreshed weekly
💕
Steph's Voice + Context
Who she is, her storefronts, her community, her tone. Affiliate disclosure. Community links. Never changes — baked into system prompt permanently.
~500 tokens · static
⚡
Current Deal Alerts
Manually curated section Steph's team updates. "This week: Target has extra 20% off kids clothing." Fast-moving deals that don't wait for the pipeline.
~200 tokens · manually updated anytime
💡 Why This Fits in a Prompt (No Database Needed Yet)
Top 20 products × ~40 words each = ~800 tokens. Full knowledge base = ~2,800 tokens. Claude's context window is 200,000 tokens. We have massive headroom — no vector database or RAG needed until Phase 3 when the catalog grows into the tens of thousands of active products. This means Phase 1 and 2 are buildable with just Python + Flask, no backend database required.

🗺 What to Build First
Phase 1 is one Python script that reads the CSVs you already have, computes the Hot Score, writes the knowledge block text file, and loads it into the chatbot system prompt. That's it. Walmart Direct API + Impact API connect in Phase 2 for live price lookups and real affiliate link generation in the content tab. URLGenius API access should be requested now — it unlocks real-time click velocity which is the strongest single signal in the whole system.
Current data alone — just from the CSVs — is enough to make the AI dramatically smarter than any generic chatbot. The APIs make it live and self-updating.
