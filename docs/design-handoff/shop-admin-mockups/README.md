# Shop and Admin Mockup Handoff

This handoff packages eight mobile-first reference mockups for the next Shop and Creator Workspace design pass.

The public pages should feel like a polished influencer recommendation experience. The admin pages should feel like one connected operations product. Use the Admin Manage mockup as the approved admin shell reference for header treatment, bottom navigation, spacing, visual status badges, and collection-first cards.

## Visual Direction

- Use the earlier theme picker concept as the north star: light neutral canvas, Sage Forward accents, refined controls, and strong product imagery.
- Keep retailer identity recognizable without letting generic marketplace colors dominate the page.
- Prefer visual product rails and collection previews over code-like text rows.
- Use `Buy Now` on public product CTAs.
- Keep bookmark affordances where they help a creator audience save recommendations.
- Avoid cart-first storefront conventions, muddy tan full-page washes, rainbow AI chips, and disconnected page-by-page styling.

## Selected Variant Alignment

- Admin Hub follows `hub_home` V1-B: urgency-first dashboard, now with the detailed attention list moved lower on the page.
- Shop Trends extends the public trending direction into two distinct shopping objects: top individual products and themed trend collections.
- Shop Posts follows the creator recommendation feed direction.
- Admin Create follows the collection editor Content / Design / Preview flow.
- EchoAgent Chat follows the selected Variant B behavior where navigation collapses while the composer has focus.

## Working Rule For The Next Design Pass

Treat these mockups as design references, not exact implementation truth. Preserve their hierarchy, interaction intent, content model, and connected visual system while adapting to the real templates and data boundaries in the app.
